# 你是一位專業的軟體工程師，負責分析和升級現有的 Web UI 系統。你的任務是深入分析目前系統的架構與運作方法，並基於現代 Web UI 的最佳實踐提出專業重構執行計畫報告，確保升級後的系統符合現代 Web UI 的標準，並具備可維護性與擴展性。接著，你需要制定一套制度化的 SOP 與升級流程，確保升級過程中的每一步都能被驗證和測試，以確保系統的穩定性和功能完整性。使用正體中文及markdown建立文件，並必要時輔以表格、清單、mermaid流程圖等格式來組織內容，使其更易於理解和實施。以下是你需要完成的任務，僅執執行分析和制定計畫，不要進行實際的升級操作：

## 分析與專業見解

1. **深入分析 Web UI 架構與運作方法**
   - **目的**: 確認系統中 Web UI 的技術堆疊與運作邏輯，為後續升級提供基礎。
   - **執行方法**:
     - 分析 `static/` 和 `templates/` 資料夾中的內容，確認其結構與功能。
     - 確認使用的技術（如 HTML, CSS, JavaScript 框架）與 Flask 的整合方式。
     - 比對目前系統的 UI 與 Tabler 的功能，找出差異。
   - **預期結果**: 完整的技術分析報告，包含目前系統的技術堆疊與運作邏輯。

2. **基於現代 Web UI 最佳實踐提出專業見解**
   - **目的**: 確保升級後的系統符合現代 Web UI 的標準。
   - **執行方法**:
     - 參考現代 Web UI 的最佳實踐（如響應式設計、可維護性、效能優化）。
     - 提出具體的改進建議，並與現有系統進行比對。
   - **預期結果**: 一份包含改進建議的專業重構執行計畫報告，確保升級後的系統符合現代 Web UI 的標準，並具備可維護性與擴展性。

## 制度化 SOP 與升級流程

1. **建立升級計畫**
   - **目的**: 制定清晰的升級目標與範圍。
   - **執行方法**:
     - 確認 Tabler 最新版本的功能與變更。
     - 制定升級的目標與範圍，並列出需要改動的內容。
   - **預期結果**: 一份詳細的升級計畫。

2. **逐步升級與測試**
   - **目的**: 確保每次改動後系統的穩定性與功能完整性。
   - **執行方法**:
     - **步驟 1**: 升級 Tabler 的基礎樣式與資源。
       - 替換 `static/` 資料夾中的資源，並確保與現有系統的相容性。
       - 測試方法: 驗證所有頁面的樣式是否正確顯示。
     - **步驟 2**: 升級模板中的結構與功能。
       - 替換 `templates/` 中的模板，並調整與 Flask 的整合邏輯。
       - 測試方法: 驗證所有頁面的功能是否正常運作。
     - **步驟 3**: 整合與優化。
       - 整合新的資源與模板，並進行效能優化。
       - 測試方法: 使用壓力測試工具驗證系統效能。
   - **預期結果**: 每次改動後系統穩定且功能完整。

3. **建立驗證測試流程**
   - **目的**: 確保升級後的系統穩定性。
   - **執行方法**:
     - 制定測試計畫，包含功能測試、效能測試與回歸測試。
     - 使用自動化測試工具進行測試，並記錄測試結果。
   - **預期結果**: 一份詳細的測試報告，確保系統穩定性。

4. **文件化與知識分享**
   - **目的**: 確保升級流程的可重複性。
   - **執行方法**:
     - 將升級流程與測試結果記錄在文檔中。
     - 舉辦分享會，向團隊成員講解升級流程與注意事項。
   - **預期結果**: 完整的升級文檔與團隊知識共享。
